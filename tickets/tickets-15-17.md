# Тикет 15: Создание компонента формы для прикрепления фотоотчета и кнопки «Завершить»

## Описание

Завершающим шагом работы монтажника на объекте является фиксация результата. Монтажник должен сфотографировать собранную мебель (шкаф или кухню) и прикрепить фото в систему. Это действие служит бизнес-триггером: заказ переводится в финальную стадию «Завершено», а бэкенд в будущем автоматически отправит клиенту чек и закрывающие документы.

## Технические требования

- **Компонент (`src/components/Installer/PhotoReportUploader.vue`):**
    - Встроить компонент на страницу деталей монтажа (из Тикета 14).
    - Использовать `<input type="file" accept="image/*" capture="environment" />`, чтобы на мобильных устройствах можно было сразу открыть камеру.
- **Предпросмотр (Preview):**
    - После выбора фотографии показывать её миниатюру на экране перед окончательной отправкой.
    - Добавить возможность удалить выбранное фото и переснять.
- **Действие завершения:**
    - Большая кнопка «Завершить монтаж» (доступна только если фото прикреплено).
    - При клике показывать спиннер загрузки, имитировать сетевой запрос (отправку `FormData` в сервис медиафайлов) и обновлять статус заказа.
- **UX/UI нюансы:**
    - После "успешной" отправки показывать полноэкранное зеленое уведомление: «Объект успешно сдан!» и возвращать пользователя к списку выездов.

## Критерии приемки (DoD)

1. Реализован загрузчик фотографий с поддержкой прямой съемки с камеры устройства.
2. Корректно работает предпросмотр выбранного снимка.
3. Кнопка «Завершить монтаж» заблокирована до момента прикрепления фото.
4. При успешном сабмите статус заказа обновляется, и монтажник видит уведомление об успешной сдаче объекта.

# Тикет 16: Разработка цифровой панели физических остатков материалов (Кладовщик)

## Описание

Рабочий экран для Кладовщика. Ему необходима цифровая панель, которая в режиме реального времени показывает физическое наличие плитных материалов и фурнитуры на складских полках. Этот экран заменяет бумажные журналы и позволяет быстро оценить, что есть в наличии.

## Технические требования

- **Страница (`src/views/Storekeeper/InventoryView.vue`):**
    - Доступ к роуту разрешен только для `userRole === 'Кладовщик'`.
    - Отображение данных: таблица номенклатуры (соответствует таблице `materials` из БД).
- **Колонки таблицы:**
    - Артикул (`article`).
    - Наименование (`name`).
    - Фактический остаток (`current_stock`).
- **Стейт / Mock-данные:**
    - Создать mock-массив складских запасов в Pinia.
    - Остатки хранятся и выводятся как числа с фиксированной точностью (имитация DECIMAL, 3 знака после запятой для метража/кубатуры, либо целые для крепежа).
- **Функционал фильтрации:**
    - Добавить строку поиска по Артикулу и Наименованию для быстрого нахождения нужной детали.

## Критерии приемки (DoD)

1. Создан защищенный роут `/storekeeper/inventory`. При логине под Кладовщиком происходит редирект на этот экран.
2. Таблица физических остатков корректно рендерится из mock-состояния.
3. Поиск по артикулу и названию фильтрует таблицу без задержек.
4. Визуальное выделение (например, красным цветом) строк, где `current_stock` меньше или равен нулю.

# Тикет 17: Реализация формы фиксации прихода новой партии фурнитуры

## Описание

Кладовщик должен своевременно вносить данные о поступивших от контрагентов (поставщиков) материалах. Это действие пополняет виртуальные остатки системы, открывая возможность запуска новых проектов в производство (так как устраняется дефицит).

## Технические требования

- **Компонент (`src/components/Storekeeper/IncomingStockModal.vue`):**
    - Модальное окно (или side-panel), вызываемое со страницы `InventoryView` по кнопке «Оформить приход».
    - **Поля формы:**
        - Выбор номенклатуры (Select или автокомплит по `article`/`name` из существующего списка материалов).
        - Количество поступившего материала (`quantity_change`) — _обязательное числовое поле (DECIMAL/Float)_.
- **Логика работы:**
    - Запретить ввод отрицательных значений в поле количества.
    - При нажатии "Сохранить" форма эмитит событие, которое увеличивает `current_stock` выбранной позиции в mock-стейте Pinia.
    - Окно автоматически закрывается, форма сбрасывается.
- **Уведомления:**
    - Показать Toast-сообщение (всплывашку): "Остатки успешно пополнены".

## Критерии приемки (DoD)

1. Сверстано модальное окно оформления прихода.
2. Выпадающий список номенклатуры корректно подтягивает доступные материалы.
3. Валидация не пропускает пустые или отрицательные значения количества.
4. При сабмите происходит реактивное обновление значения `current_stock` в главной таблице (Тикет 16), окно закрывается, выводится уведомление об успехе.
