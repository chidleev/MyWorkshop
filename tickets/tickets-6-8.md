# Тикет 6: Создание канбан-доски / таблицы для раздела «Мои заказы» (Менеджер)

## Описание

Необходимо разработать главный рабочий экран Менеджера — раздел «Мои заказы» (согласно Use-Case: "Отслеживание прогресса заказа"). Экран должен наглядно отображать реестр активных сделок с их текущими статусами готовности, чтобы менеджер мог контролировать сроки и давать обратную связь клиентам.

## Технические требования

- **Компонент (`src/views/Manager/OrdersView.vue`):**
    - Реализовать отображение списка заказов. Предпочтительный формат — **Канбан-доска** (колонки по этапам: «В обработке», «Распил», «Кромление», «Сборка», «Готов к отгрузке»), либо Data Table с цветовым кодированием статусов (badges).
    - Каждая карточка заказа должна отображать: Номер договора, ФИО клиента, Плановую дату сдачи (`target_date`), Итоговую стоимость (если уже рассчитана).
- **Стейт / API:**
    - Создать заглушку (mock-данные) массива заказов. Структура должна соответствовать таблице `orders` из физической модели БД.
    - Использовать `onMounted` для имитации асинхронной загрузки данных.
- **Стилизация:**
    - Использовать Tailwind CSS (или выбранный UI-фреймворк) для создания сетки колонок/таблицы. Карточки должны быть читаемыми, с акцентом на просроченные (или близкие к дедлайну) заказы.

## Критерии приемки (DoD)

1. Компонент `OrdersView.vue` создан и доступен по соответствующему роуту (только для роли "Менеджер").
2. Данные успешно загружаются из mock-массива и корректно распределяются по статусам.
3. Реализована цветовая индикация для карточек (например, желтый бейдж — в работе, зеленый — готов).
4. Интерфейс адаптируется под размеры экрана.

# Тикет 7: Реализация модального окна / страницы создания новой карточки заказа

## Описание

Менеджеру нужен интерфейс для заведения новой сделки в системе. На этом этапе фиксируются первичные контактные данные клиента и плановые сроки. Спецификация (CSV) загружается позже, в рамках отдельного тикета.

## Технические требования

- **Компонент (`src/components/Orders/OrderFormModal.vue`):**
    - Создать переиспользуемое модальное окно (или side-panel/выдвижную панель) с формой.
    - **Поля формы:**
        - ФИО клиента (`full_name`) — _обязательное_
        - Номер телефона (`phone`) — _обязательное, с маской ввода_
        - E-mail (`email`)
        - Адрес объекта (`address`) — _текстовое поле_ (для монтажников)
        - Номер договора (`agreement_number`) — _обязательное_
        - Плановый срок сдачи (`target_date`) — _выбор даты (datepicker), не может быть в прошлом_
- **Валидация:** Реализовать базовую валидацию на стороне клиента (например, через VeeValidate или простые вычисляемые свойства `computed`).
- **Логика работы:**
    - При нажатии "Создать", форма должна эмитить (emit) событие наверх или вызывать mock-экшен в Pinia для добавления нового заказа в стейт.
    - После успешного добавления модалка закрывается, форма очищается.

## Критерии приемки (DoD)

1. Разработано модальное окно с формой и всеми требуемыми полями ввода.
2. Настроена валидация: форма не отправляется, если не заполнены обязательные поля.
3. При сабмите происходит добавление карточки в общий список (на канбан-доске из Тикета 6 моментально появляется новая карточка в статусе «В обработке» / «Новый»).
4. Окно корректно закрывается, стейт формы сбрасывается.

# Тикет 8: Реализация функционала редактирования и удаления карточки заказа

## Описание

Сделка может быть отменена, или клиент может попросить изменить контактные данные/сроки. Необходимо дать менеджеру инструмент для корректировки информации в уже созданной карточке заказа.

## Технические требования

- **UI элементы:**
    - Добавить в карточку заказа (на канбан-доске или в таблице) выпадающее меню (dropdown) или иконки действий: «Редактировать» и «Удалить» (или «Отменить сделку»).
- **Редактирование:**
    - Использовать компонент `OrderFormModal.vue` из Тикета 7.
    - Передавать в него prop (например, `orderId` или объект `initialData`), чтобы при открытии поля заполнялись текущими данными заказа.
    - При сабмите вызывать mock-экшен PUT/PATCH для обновления карточки в стейте.
- **Удаление:**
    - При клике на «Удалить» показывать окно подтверждения (Confirmation Dialog): _"Вы уверены, что хотите удалить заказ[Номер договора]? Это действие необратимо."_
    - При подтверждении — удалять заказ из списка (mock DELETE запрос).

## Критерии приемки (DoD)

1. Форма `OrderFormModal.vue` успешно переиспользуется для режима редактирования (поля предзаполняются).
2. Изменения корректно сохраняются и сразу отображаются в интерфейсе.
3. Добавлено диалоговое окно подтверждения удаления.
4. При удалении заказ реактивно пропадает с канбан-доски.
