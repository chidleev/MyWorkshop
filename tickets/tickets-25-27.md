# Тикет 25: Развертывание СУБД MySQL и создание пустой базы данных

## Описание

Первым шагом в реализации серверной части является подготовка инфраструктуры для хранения данных. Согласно архитектурным требованиям, используется реляционная СУБД MySQL с движком InnoDB для обеспечения надежности транзакций (ACID).

## Технические требования

- **Среда развертывания:**
    - Использовать `docker-compose` для локальной разработки (образ `mysql:8.0` или выше), либо установить локально.
    - Настроить переменные окружения: `MYSQL_ROOT_PASSWORD`, `MYSQL_DATABASE`, `MYSQL_USER`, `MYSQL_PASSWORD`.
- **База данных:**
    - Создать пустую БД с названием `my_workshop`.
    - Установить кодировку `utf8mb4` и сопоставление (collation) `utf8mb4_unicode_ci` для корректной поддержки кириллицы и спецсимволов.
- **Управление миграциями (опционально, но рекомендуется):**
    - Начать формирование папки `migrations/` или SQL-дампа `init.sql` для идемпотентного наката структуры БД.

## Критерии приемки (DoD)

1. Написан файл `docker-compose.yml` (если выбрана контейнеризация) для быстрого поднятия БД.
2. База данных `my_workshop` успешно создана.
3. Доступ к базе данных проверен через любой клиент (DBeaver, DataGrip, MySQL Workbench).
4. Базовый движок по умолчанию для таблиц — InnoDB.

# Тикет 26: Настройка таблиц: создание таблицы `clients` (Клиенты)

## Описание

Реализация первой таблицы из физической модели базы данных (согласно п. 2.7.1.1 спецификации). Таблица предназначена для хранения контактной информации конечных потребителей услуг.

## Технические требования

- **Таблица `clients`:**
    - Создать таблицу с использованием движка `InnoDB`.
- **Структура полей:**
    - `id` — `INT`, Первичный ключ (Primary Key), Автоинкремент (Auto Increment). Уникальный идентификатор заказчика.
    - `full_name` — `VARCHAR(255)`, `NOT NULL`. Полное имя клиента.
    - `phone` — `VARCHAR(50)`. Номер контактного телефона.
    - `email` — `VARCHAR(255)`. Адрес электронной почты.
    - `address` — `TEXT`. Физический адрес объекта монтажа.
- **Индексы:**
    - Первичный индекс по полю `id`.

## Критерии приемки (DoD)

1. Написан и выполнен SQL-скрипт (или файл миграции) для создания таблицы `clients`.
2. Типы данных и ограничения длины (`VARCHAR(255)`, `VARCHAR(50)`) строго соблюдены.
3. Таблица успешно создана в базе данных `my_workshop`.

# Тикет 27: Настройка таблиц: создание таблицы `orders` (Заказы)

## Описание

Создание таблицы `orders`, которая выступает центральным узлом учета всех сделок предприятия (п. 2.7.1.2 спецификации). В ней хранятся плановые сроки, итоговая стоимость мебели и идентификаторы сотрудников (менеджеров).

## Технические требования

- **Таблица `orders`:**
    - Движок: `InnoDB`.
- **Структура полей:**
    - `id` — `INT`, Первичный ключ (PK), Автоинкремент. Уникальный номер карточки проекта.
    - `client_id` — `INT`, `NOT NULL`. Внешний ключ (Foreign Key) на таблицу `clients(id)` с ограничением (ON DELETE RESTRICT / CASCADE).
    - `manager_ext_id` — `VARCHAR(255)`, `NOT NULL`. Внешний идентификатор менеджера из корпоративной CRM.
    - `agreement_number` — `VARCHAR(100)`. Официальный номер бумажного договора.
    - `created_at` — `DATETIME`, По умолчанию текущее время (`DEFAULT CURRENT_TIMESTAMP`). Точная дата регистрации обращения.
    - `target_date` — `DATE`. Плановый срок финальной сдачи изделия.
    - `total_cost` — `DECIMAL(10,2)`. Итоговая стоимость реализации мебели. _Важно: Использовать именно DECIMAL для исключения математических ошибок округления._
    - `current_stage` — `VARCHAR(100)`, `NOT NULL`. Актуальный этап производственного цикла.
- **Индексы и связи:**
    - Добавить индекс по `client_id` и явно задать `FOREIGN KEY` ограничение связи с таблицей `clients`.

## Критерии приемки (DoD)

1. Написан и выполнен SQL-скрипт (или миграция) для создания таблицы `orders`.
2. Поле `total_cost` имеет формат `DECIMAL` с точностью 10 и 2 знаками после запятой.
3. Связь Foreign Key с таблицей `clients` корректно настроена в базе данных.
4. Ограничения `NOT NULL` и `DEFAULT CURRENT_TIMESTAMP` отрабатывают корректно при вставке тестовой записи.
