# Тикет 12: Создание UI-компонента карточки сменного задания с кнопками переключения статусов

## Описание

В продолжение Тикета 11, карточка сменного задания должна быть интерактивной. Мастеру цеха нужно иметь возможность в один клик перевести задание из статуса «Ожидает» в «В работе», а затем — в «Завершено». Согласно бизнес-логике, перевод задачи в активную стадию («В производстве») является важнейшим триггером, который позже будет запускать автоматическое списание материалов.

## Технические требования

- **Компонент (`src/components/Workshop/ShiftTaskCard.vue`):**
    - Принимает через `props` объект задания (`task`), содержащий id, order_id, operation_name, progress_status.
    - Реализовать блок кнопок действий, меняющийся в зависимости от `progress_status`:
        - Если статус "Новая" — кнопка "Взять в работу" (primary цвет).
        - Если статус "В работе" — кнопка "Завершить операцию" (success цвет).
        - Если статус "Завершено" — иконка галочки/текст "Выполнено" (кнопки скрыты или задизейблены).
- **События (Emits) и Логика:**
    - При нажатии на кнопку компонент должен эмитить событие `update-status` с новым значением статуса.
    - Родительский компонент (`ShiftTasksView.vue`) должен перехватывать это событие и обновлять mock-стейт, имитируя отправку сетевой команды на бэкенд.
- **UX/UI нюансы:**
    - Кнопки должны быть большими (min-height: 48px), чтобы мастеру было удобно нажимать их в перчатках или на ходу с планшета.

## Критерии приемки (DoD)

1. Компонент `ShiftTaskCard.vue` изолированно сверстан и интегрирован в список из Тикета 11.
2. Кнопки смены статуса корректно отображаются в зависимости от текущего состояния задачи.
3. При нажатии на кнопку статус реактивно обновляется в UI.
4. Интерфейс адаптирован под touch-устройства (отсутствуют мелкие элементы управления).

# Тикет 13: Реализация экрана списка назначенных выездов (Монтажник)

## Описание

Монтажник работает "в полях", поэтому его интерфейс должен быть ориентирован на мобильные устройства (Mobile-First). Перед выездом монтажник открывает приложение на телефоне или планшете, чтобы увидеть список адресов и заказов, назначенных ему на сегодня.

## Технические требования

- **Страница (`src/views/Installer/DeploymentsView.vue`):**
    - Доступ к роуту разрешен только для `userRole === 'Монтажник'`.
    - Сверстать список карточек выездов (опираясь на данные из таблицы `orders` со статусом, близким к отгрузке/монтажу).
- **Содержимое карточки списка:**
    - Имя клиента (`full_name`).
    - Краткий адрес.
    - Плановое время/срок установки.
    - Статус (например, "Ожидает выезда").
- **Стейт / Mock-данные:**
    - Создать мок-массив заказов, назначенных на текущего пользователя (имитация ответа `Контроллера заказов` для монтажника).

## Критерии приемки (DoD)

1. Создан защищенный роут `/installer/deployments`. При логине под Монтажником происходит редирект на этот экран.
2. Список выездов отображается корректно и берет данные из mock-стора.
3. Верстка строго Mobile-First (карточки занимают всю ширину экрана на смартфонах, шрифты читаемы на улице).
4. Каждая карточка кликабельна (подготовка к переходу на экран деталей из Тикета 14).

# Тикет 14: Разработка карточки деталей монтажа (адрес, рендеры, сроки)

## Описание

При клике на конкретный выезд в списке, монтажнику открывается детальная карточка. Здесь он может посмотреть точный адрес, контактный телефон клиента для связи, а главное — графическую визуализацию (рендеры) готового изделия, чтобы понимать, как должна выглядеть собранная мебель.

## Технические требования

- **Страница (`src/views/Installer/DeploymentDetailView.vue`):**
    - Переход по роуту с параметром, например `/installer/deployments/:id`.
    - **Информационный блок:** ФИО клиента, кликабельный телефон (ссылка `tel:`), полный адрес (`address`).
    - **Галерея рендеров:** Блок для просмотра изображений (связь с таблицей `media_files` из БД).
- **Функционал просмотра изображений:**
    - Реализовать простую сетку изображений. При клике на картинку она должна открываться на весь экран (Lightbox / Modal).
    - Использовать mock-ссылки (на внешние плейсхолдеры) для имитации загруженных дизайн-проектов.
- **Навигация:**
    - В шапке (Header) или интерфейсе должна быть явная кнопка «Назад» для возврата к списку выездов.

## Критерии приемки (DoD)

1. Роут детализации корректно отрабатывает и загружает данные нужного заказа по ID.
2. Контактная информация отображается, телефон кликабелен для быстрого звонка клиенту.
3. Изображения (рендеры) отображаются в виде сетки/карусели.
4. Реализован механизм просмотра картинок в увеличенном виде (Lightbox).
5. Присутствует кнопка возврата на предыдущий экран.
