# Тикет 77: Настройка CORS и подготовка окружения (Environment) для Production

## Описание

Перед выгрузкой проекта на боевой сервер необходимо защитить API от сторонних запросов и убедиться, что фронтенд и бэкенд смогут беспрепятственно общаться друг с другом по защищенному протоколу HTTPS. Настройки локальной разработки (где разрешен любой CORS) не подходят для продакшена.

## Технические требования

- **Безопасность Бэкенда (`src/app.ts`):**
    - Установить и подключить пакет `helmet` для базовой защиты HTTP-заголовков.
    - Настроить `cors`: разрешить запросы _только_ с конкретного домена, на котором будет размещен фронтенд (взять значение из `process.env.FRONTEND_URL`).
- **Конфигурация переменных окружения:**
    - Создать шаблоны `.env.example` для фронтенда и бэкенда, чтобы при деплое на Serv00 было понятно, какие ключи нужно заполнить.
    - _Бэкенд:_ `PORT`, `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `FRONTEND_URL`, `EXTERNAL_API_KEY`, `CRM_API_URL`.
    - _Фронтенд:_ Убедиться, что вызовы API идут через переменную `import.meta.env.VITE_API_URL`.
- **Проверка путей (Paths):**
    - Убедиться, что пути к папке `uploads/` (где лежат фотоотчеты монтажников) формируются динамически (через `__dirname` или `process.cwd()`), так как файловая структура на Serv00 (FreeBSD) будет отличаться от локальной.

## Критерии приемки (DoD)

1. Пакет `helmet` установлен и активирован.
2. CORS-политика на бэкенде разрешает запросы только с указанного в `.env` домена.
3. Добавлены файлы `.env.example` в обе части проекта (клиент и сервер).
4. Проект готов к сборке и переносу на хостинг.

# Тикет 78: Деплой приложения на хостинг Serv00.com (Frontend + Backend + MySQL)

## Описание

Финальный шаг — выгрузка рабочей системы на бесплатный FreeBSD-хостинг Serv00.com. Так как платформа использует собственную панель (Devil) и Phusion Passenger для запуска Node.js (вместо Docker/PM2), необходимо развернуть базу данных, бэкенд и статику фронтенда по правилам хостинга.

## Технические требования

- **Шаг 1: База данных (MySQL):**
    - В панели Serv00 (Databases -> MySQL) создать базу данных и пользователя.
    - Экспортировать локальную структуру БД (дамп таблиц из Тикетов 26-33) и импортировать её на сервере через phpMyAdmin или консоль SSH.
- **Шаг 2: Бэкенд (Node.js):**
    - Скомпилировать TypeScript-код бэкенда в JavaScript (команда `npm run build` -> генерация папки `dist`).
    - В панели Serv00 (WWW websites -> Add website) создать сайт с типом **Node.js**.
    - По SSH или через File Manager загрузить содержимое папки `dist` и файлы `package.json`, `.env` в директорию сайта (обычно `public_nodejs`).
    - Подключиться по SSH, зайти в папку сайта и выполнить `npm install --production`.
    - _Специфика Serv00:_ Убедиться, что точкой входа является файл `app.js` (Phusion Passenger ищет именно его). Если после билда главный файл называется `index.js`, переименовать его или настроить конфигурацию.
- **Шаг 3: Фронтенд (Vue.js + Apache/LiteSpeed):**
    - В локальном проекте фронтенда прописать в `.env.production`: `VITE_API_URL=https://<домен-вашего-бэкенда-на-serv00>`.
    - Выполнить сборку Vue: `npm run build` (генерируется папка `dist`).
    - В панели Serv00 создать стандартный сайт (тип **PHP/Static**). Выпустить бесплатный SSL-сертификат (SSL -> Let's Encrypt).
    - Загрузить содержимое фронтенд-папки `dist` в директорию `public_html` этого сайта.
    - _Настройка SPA-маршрутизации:_ Внутри `public_html` создать файл `.htaccess` со следующим кодом для перенаправления всех запросов на `index.html` (обязательно для Vue Router):
    ```apache
    <IfModule mod_rewrite.c>
      RewriteEngine On
      RewriteBase /
      RewriteRule ^index\.html$ - [L]
      RewriteCond %{REQUEST_FILENAME} !-f
      RewriteCond %{REQUEST_FILENAME} !-d
      RewriteRule . /index.html [L]
    </IfModule>
    ```

## Критерии приемки (DoD)

1. БД MySQL успешно развернута на Serv00 и наполнена таблицами.
2. Бэкенд запущен через Phusion Passenger, отвечает по HTTPS и успешно подключается к БД.
3. Статика Vue.js загружена в `public_html`, HTTPS активен.
4. При перезагрузке страниц фронтенда (F5) не возникает ошибки 404 Not Found (работает `.htaccess`).
5. Проведен тестовый Smoke-тест на боевом сервере (успешно создан тестовый заказ).
