# Тикет 1: Инициализация проекта Vue.js (TypeScript, Composition API, сборщик)

## Описание

Первым шагом в разработке пользовательского интерфейса (согласно архитектуре C4, уровень 2 "Контейнеры") является создание базового каркаса SPA-приложения (Single Page Application). Проект должен быть настроен с использованием современных стандартов экосистемы Vue 3 для обеспечения высокой производительности, легковесности и читаемости кода.

## Технические требования

- **Сборщик и Фреймворк:**
    - Инициализировать проект с помощью бандлера Vite (`npm create vite@latest ... --template vue-ts`).
    - Версия Vue — 3.x.
- **Подход к компонентам:**
    - Строго использовать Composition API с синтаксисом `<script setup>`. Отказаться от устаревшего Options API.
- **Типизация:**
    - Проект должен быть написан на TypeScript.
    - В `tsconfig.json` включить строгий режим (`"strict": true`).
- **Структура директорий:**
    - Создать стандартизированную архитектуру папок внутри `src/`:
        - `api/` — конфигурация Axios и сервисы для отправки REST-запросов к бэкенду.
        - `assets/` — статичные файлы, шрифты, базовые CSS-файлы.
        - `components/` — изолированные и переиспользуемые UI-компоненты (кнопки, инпуты, модалки, глобальные оболочки вроде `GlobalToasts.vue`, общие таблицы вроде `components/Common/AppDataTable.vue`).
        - `layouts/` — каркасы для оборачивания страниц (например, `AppLayout.vue`).
        - `stores/` — модули глобального состояния Pinia (в т.ч. `auth`, `notifications` для тостов).
        - `views/` — полноценные экраны (страницы приложения).
        - `utils/` — вспомогательные функции (форматирование валюты, дат, `datetime.ts`, телефон `phone.ts`, хелперы маршрутов ролей).
- **Контроль качества кода (Code Quality):**
    - Установить и настроить **ESLint** (с официальным плагином `eslint-plugin-vue` для Vue 3).
    - Установить **Prettier** для автоматического форматирования. Убедиться, что они не конфликтуют (использовать `eslint-config-prettier`).
    - Добавить npm-скрипты `lint` и `format` в `package.json`.

## Критерии приемки (DoD)

1. Исходный код сгенерирован, зависимости устанавливаются без конфликтов (`npm install` / `pnpm install`).
2. Проект успешно запускается локально командой `npm run dev` и отображает дефолтную стартовую страницу.
3. Проект успешно компилируется для продакшена (`npm run build`), TypeScript не выдает ошибок типизации.
4. Вызов `npm run lint` корректно проверяет `.vue` и `.ts` файлы, не выдавая критических ошибок.
5. Создана и зафиксирована в коммите базовая структура папок `src/`.
